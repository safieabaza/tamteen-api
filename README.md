# Tamteen API

A production-ready Laravel 11 REST API for Tamteen — a CV builder with real ATS (Applicant Tracking System) analysis scoring.

## Features

- **OTP Authentication** — Passwordless login via email with 6-digit codes
- **CV Management** — Full CRUD for structured CV data (experience, education, skills)
- **ATS Analysis** — Real 4-dimension scoring algorithm (no AI required)
- **JWT Auth** — 7-day Bearer tokens via `tymon/jwt-auth`
- **Rate Limiting** — 3 OTP requests per hour per email
- **Beautiful Email** — Gradient HTML OTP email template

## Installation

### 1. Install dependencies
```bash
composer install
```

### 2. Configure environment
```bash
cp .env.example .env
php artisan key:generate
php artisan jwt:secret
```

Edit `.env` — configure your database and Mailtrap credentials (see below).

### 3. Database setup
```bash
# SQLite (default, no server needed)
touch database/database.sqlite

php artisan migrate
php artisan db:seed --class=ATSKeywordSeeder
```

### 4. Start the server
```bash
php artisan serve
# → http://localhost:8000
```

### Auth Endpoints

#### POST `/api/auth/send-otp`
Send a 6-digit OTP to the user's email.

**Rate limit:** 3 requests per hour per email

**Request:**
```json
{ "email": "user@example.com" }
```

**Response (200):**
```json
{
  "success": true,
  "message": "Verification code sent to your email",
  "data": { "email": "user@example.com" }
}
```

**Error (429 — rate limited):**
```json
{
  "success": false,
  "message": "Too many OTP requests. Please wait 1 hour before trying again.",
  "data": []
}
```

---

#### POST `/api/auth/verify-otp`
Verify the OTP and receive a JWT token.

**Request:**
```json
{
  "email": "user@example.com",
  "code": "482910"
}
```

**Response (200):**
```json
{
  "success": true,
  "message": "Email verified successfully",
  "data": {
    "token": "eyJ0eXAiOiJKV1QiLCJhbGciOiJIUzI1NiJ9...",
    "token_type": "Bearer",
    "expires_in": 604800,
    "user": {
      "id": 1,
      "email": "user@example.com",
      "is_verified": true
    }
  }
}
```

### CV Endpoints

#### GET `/api/cvs` 🔒
List all CVs for the authenticated user.

#### POST `/api/cvs` 🔒
Create a new CV.

**Request body:**
```json
{
  "title": "Senior Engineer CV",
  "full_name": "John Doe",
  "email": "john@example.com",
  "phone": "+1-555-0123",
  "summary": "Experienced engineer with 7+ years...",
  "experience": [
    {
      "title": "Senior Engineer",
      "company": "TechCorp",
      "start_date": "2020-01",
      "end_date": "Present",
      "description": [
        "Led team of 5 engineers, improving velocity by 25%",
        "Implemented microservices reducing latency by 35%"
      ]
    }
  ],
  "education": [
    {
      "degree": "BSc Computer Science",
      "institution": "University of Technology",
      "year": "2017"
    }
  ],
  "skills": ["JavaScript", "Python", "React", "AWS", "Docker"]
```

#### GET `/api/cvs/{id}` 🔒
Get a single CV with its latest analysis.

#### PUT `/api/cvs/{id}` 🔒
Update a CV (partial updates supported).

#### DELETE `/api/cvs/{id}` 🔒
Delete a CV and all its analyses.

### ATS Analysis Endpoints

#### POST `/api/cvs/{id}/analyze` 🔒
Run ATS analysis on a CV and get a score 0-100.

**Response:**
```json
{
  "success": true,
  "message": "ATS analysis completed",
  "data": {
    "analysis_id": 1,
    "cv_id": 1,
    "ats_score": 78.5,
    "score_breakdown": {
      "keyword_match":   { "score": 28.3, "max": 40 },
      "formatting":      { "score": 18.0, "max": 20 },
      "content_quality": { "score": 17.0, "max": 20 },
      "completeness":    { "score": 15.2, "max": 20 }
    },
    "matched_keywords": [
      { "keyword": "javascript", "category": "tech_skill", "occurrences": 2 }
    ],
    "missing_keywords": [
      { "keyword": "kubernetes", "category": "tech_skill", "weight": 1.5 }
    ],
    "recommendations": [
      "Add quantifiable achievements (e.g., increased revenue by 30%)",
      "Consider adding these high-impact keywords: kubernetes, devops, ci/cd"
    ]
  }
}
```

#### GET `/api/cvs/{id}/analyses` 🔒
Get the full analysis history for a CV.

## ATS Scoring Algorithm

### 1. Keyword Match (40 points)
- Extracts all text from summary, experience, education, and skills
- Scans against 75 keywords in the `ats_keywords` table
- Each keyword has a weight (0.8-1.5) based on industry demand
- Diminishing returns: max benefit at 2 occurrences
- Score = `(matched_weight / total_weight) * 40`

### 2. Formatting (20 points)
| Check | Points |
|-------|--------|
| Has experience + education + skills sections | +5 |
| Complete contact info (name, email, phone) | +5 |
| Reasonable length (200-1000 words) | +4 |
| Bullet points in experience | +3 |
| Consistent formatting (title + company per entry) | +3 |

### 3. Content Quality (20 points)
| Check | Points |
|-------|--------|
| Quantifiable achievements (%, $, numbers) | +8 |
| Detailed descriptions (>50 chars per entry) | +6 |
| Professional summary (20+ words) | +3 |
| Complete contact details | +3 |

### 4. Completeness (20 points)
| Field | Points |
|-------|--------|
| experience | +5 |
| summary | +4 |
| full_name | +3 |
| email | +3 |
| education | +3 |
| phone | +2 |

## Database Schema

```
users          (id, email, is_verified, timestamps)
otps           (id, email, code, expires_at, verified_at, timestamps)
cvs            (id, user_id, title, full_name, email, phone, summary,
                experience JSON, education JSON, skills JSON, ats_score, timestamps)
ats_keywords   (id, category, keyword, weight, timestamps)
cv_analyses    (id, cv_id, ats_score, matched_keywords JSON,
                missing_keywords JSON, recommendations JSON,
                score_breakdown JSON, timestamps)
```

## Testing with Postman

1. Import `postman_collection.json` into Postman
2. Set `base_url` variable to `http://localhost:8000/api`
3. **Send OTP** → check Mailtrap inbox for the code
4. **Verify OTP** with the code → token auto-saved to `{{token}}`
5. **Create CV** → cv_id auto-saved to `{{cv_id}}`
6. **Analyze CV** → see ATS score breakdown and recommendations
